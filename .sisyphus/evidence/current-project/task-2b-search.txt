## Schema Notes

  attachments PRIMARY KEY: (stored_path, email_stable_id)
  → Multiple emails containing the same file get separate rows
  → stored_path points to the same file on disk (deduped by sha256_size.ext)
  → email_stable_id links each row to its email in the emails table

  emails.stable_id referenced by attachments.email_stable_id:
  → JOIN possible: find all emails containing a specific attachment
  → Reverse JOIN possible: find all attachments for a given email

## Example JOIN Query

  SELECT e.mailbox, e.from_addr, e.subject, e.date, a.original_filename, a.stored_path
  FROM attachments a
  JOIN emails e ON a.email_stable_id = e.stable_id
  WHERE a.original_filename LIKE '%quarterly%'
  ORDER BY e.date;
