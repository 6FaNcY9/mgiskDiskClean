# mgiskDiskClean

Self-contained devenv for mrija.org mailbox disk cleanup.
Scans Maildir attachments and serves an HTML report for the owner to review.

## Structure

```
mgiskDiskClean/
  devenv.nix          # devenv shell with all packages + commands
  devenv.yaml         # nixpkgs input pins
  nginx/
    nginx.conf        # nginx config — serves on port 8765
    mime.types        # mime types for nginx
    .htpasswd         # boss login (generated by set-password, not in git)
  reports/
    index.html        # deployed report (generated, not in git)
  scripts/
    maildir_attachments_py2.py   # scanner script
  logs/               # nginx logs (generated, not in git)
```

## Setup

```bash
# 1. enter devenv
devenv shell

# 2. set boss password
set-password

# 3. scan a mailbox on the server and pull report
ssh mrija_org@s16.thehost.com.ua \
  "python maildir_attachments_py2.py email/mrija.org/andrii.karioti/.maildir/.Sent"
scp mrija_org@s16.thehost.com.ua:/email/mrija.org/andrii.karioti/attachments_report.html .

# 4. deploy report
deploy-report ./attachments_report.html

# 5. start nginx
serve-start
```

Boss opens: http://<your-ip>:8765

## Workflow for each mailbox

```bash
# scan on server
ssh mrija_org@s16.thehost.com.ua \
  "python maildir_attachments_py2.py email/mrija.org/<mailbox>/.maildir"

# pull and deploy
scp mrija_org@s16.thehost.com.ua:/email/mrija.org/<mailbox>/attachments_report.html .
deploy-report ./attachments_report.html

# reload so boss sees new report
serve-reload
```

## Commands

| Command           | Description                              |
|-------------------|------------------------------------------|
| `scan <path>`     | Run attachment scanner on a Maildir path |
| `deploy-report`   | Copy HTML report to serve directory      |
| `set-password`    | Set/reset boss login password            |
| `serve-start`     | Start nginx on port 8765                 |
| `serve-stop`      | Stop nginx                               |
| `serve-reload`    | Reload nginx (no downtime)               |

## Security notes

- nginx is password protected (HTTP Basic Auth)
- Only serves the reports/ directory — no other paths exposed
- Dotfiles blocked
- No caching — boss always sees the latest report
- Run behind your DDNS (vinohome.dedyn.io) or just locally on LAN
- Do NOT commit .htpasswd or reports/ to git
